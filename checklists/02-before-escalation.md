\# Before Escalation Checklist



\## 1) Scope confirmed

\- Issue scope clearly identified (user / device / site / org-wide)

\- Reproduced or verified by engineer



\## 2) Network baseline completed

\- Default gateway reachable

\- DNS resolution tested

\- Required ports tested

\- Results documented (PASS / FAIL / INFO)



\## 3) Identity \& system checks

\- Device time synchronized

\- Secure channel verified (if domain-joined)

\- Group memberships confirmed (if auth-related)

\- Recent changes reviewed (patches, firewall, GPO, VPN)



\## 4) Evidence ready

\- Commands run and outputs saved

\- Error messages/screenshots captured

\- Timeline of events documented



\## 5) Escalation decision

\- Escalate only if:

&nbsp; - Root cause is external

&nbsp; - Requires elevated access

&nbsp; - Requires architectural change
